# 전체 시스템 아키텍처

- 사용자가 질문하면 그와 관련된 문서를 Elasticsearch라는 데이터베이스에서 찾아내고, 찾아낸 문서의 내용을 사용자의 질문과 함께 LLM에 전달해서 답변 생성.

## 전체 동작 흐름

1. 데이터 저장 (Embedding)- 사용자가 문서를 제공하면, 문서를 의미 있는 작은 조각으로 나눈다. 
- 각 조각을 숫자로 된 벡터로 변환해서 Elasticsearch에 저장한다.

2. 질문 처리 (Query):
- 사용자가 질문하면, 질문도 벡터로 변환한다.
- Elasticsearch에서 질문 벡터와 가장 유사한 문서 조각들을 검색(Retrieval)한다.
- 검색된 문서 내용과 사용자 질문을 미리 정의된 프롬프트에 넣어서 LLM에게 전달한다.

3. API 서버
Flask 이용해서 API 형태로 제공한다.
